Detailed Description

Central event queue manager for processing input events.

The EventManager class provides a centralized system for managing input events from various sources (keyboard, mouse, console). It offers multiple consumption patterns:

  • Polling: Individual event retrieval with pollEvent()
  • Callback-based: Automatic event dispatch to type-specific handlers
  • Inspection: Non-consuming event queue examination

Events are processed in FIFO order and can be consumed by applications or handled automatically through callback registration. The manager tracks processing state and provides efficient event removal after handling.

Thread safety: This class is not thread-safe and should be used from a single thread (typically the main application thread).

Process events using type-specific callback functions.

Template Parameters
CallbacksVariadic template for callback function types
Parameters
callbacksFunctions to handle specific event types

Advanced event processing system that automatically dispatches events to appropriate callback functions based on event type. Callbacks are invoked with the event type marker and event data. Processed events are automatically removed from the queue.

Supported callback signatures:

  • void callback(KeyPressEvent, const Event&)
  • void callback(KeyReleaseEvent, const Event&)
  • void callback(MouseMoveEvent, const Event&)
  • void callback(MouseScrollEvent, const Event&)
  • void callback(ConsoleEvent, const Event&)
  • void callback(InvalidEvent, const Event&)

Example usage: cpp eventManager.handleEvents( [](KeyPressEvent, const Event& e) { std::cout << "Key pressed: " << static_cast<int>(e.key) << std::endl; }, [](MouseMoveEvent, const Event& e) { std::cout << "Mouse moved: " << e.mouseDelta.x << ", " << e.mouseDelta.y << std::endl; } );

◆ pollEvent()

std::optional< Event > til::EventManager::pollEvent ( )

Poll and consume next event from queue.

Returns
Optional containing next event, or empty if no events available

Retrieves and removes the next event from the front of the queue. Returns std::nullopt when no events are pending. This provides manual event consumption for applications that need direct control over event processing timing.

Example usage: cpp while (auto event = eventManager.pollEvent()) { if (event->isOfType<KeyPressEvent>()) { handleKeyPress(event->key); } }
