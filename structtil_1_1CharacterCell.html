Detailed Description

Structure representing a single character cell in terminal display.

A character cell is the fundamental unit of terminal text display, containing both the character to display and its visual properties. Terminal displays are organized as a 2D grid of these cells.

The structure uses Unicode codepoints to support international text and symbols beyond basic ASCII. Color information allows for rich visual presentation within terminal constraints.

Default values create a visible placeholder (hash symbol '#') in magenta color for easy identification of uninitialized cells.

◆ codepoint

u32 til::CharacterCell::codepoint = 35

Unicode codepoint of the character to display.

Represents the character using Unicode codepoint values. Defaults to 35 (ASCII '#' - hash/pound symbol) for easy identification of default/uninitialized cells.

Common values:

  • 32: Space character (invisible)
  • 35: Hash symbol '#' (default)
  • 65-90: Uppercase letters A-Z
  • 97-122: Lowercase letters a-z
  • Values > 127: Extended Unicode characters

Color til::CharacterCell::color = Color{128, 0, 128}

Color information for character rendering.

Specifies the visual appearance of the character including foreground color and transparency. The exact interpretation depends on terminal capabilities and rendering mode.

Defaults to Color{128, 0, 128} - a medium magenta color that stands out for debugging uninitialized cells.
